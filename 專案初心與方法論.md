# 專案初心與方法論

## 專案定位

`check_price` 不是預言市場的 AI，也不是保證獲利的自動交易系統。

它的定位是：

- 把盤面拆成可理解的事件與條件
- 把交易流程拆成可執行的模塊
- 用資料與回測持續修正判斷
- 幫交易者降低情緒干擾，而不是取代交易者

一句話說明：

> `check_price` 是一套把交易拆成「進場、出場、風控、驗證」四個模塊的 AI 決策輔助系統。

## 專案初心

這個專案的初心不是「每次猜對漲跌」，而是建立一個可以持續學習的決策迴路：

1. 先用資料理解市場
2. 再把市場拆成事件
3. 再把事件轉成可執行的方案
4. 最後用 live 成效與回測持續修正

核心不是神預測，而是：

- 執行一致性
- 風險透明
- 可回頭驗證
- 可持續修正

## 方法論主軸

### 1. 進場

專案不直接把「方向」當成交易，而是把進場拆成事件層級：

- `approach_up / approach_down`
  - 戰備層
  - 作用是提醒你進入觀察，不是直接下單

- `breakout_touch_up / breakout_touch_down`
  - 試單層
  - 作用是提醒第一次碰到關鍵位，但還沒完成確認

- `effective_long_breakout / effective_short_breakdown`
  - 首層確認
  - 作用是告訴你市場已經有第一層條件成立

- `retest_hold_*`
  - setup 層
  - 作用是確認劇本是否延續，不保證是最佳第一槍

- `second_breakout_* / second_breakdown_*`
  - 保守掛單 / 確認加碼層
  - 作用是提供更保守的確認點，不預設它永遠是最佳首次進場

### 2. 出場

專案的出場不是只有一條停利線，而是完整的管理方案：

- `stop_loss`
- `take_profit_1`
- `take_profit_2`
- `breakeven_trigger`
- `breakeven_stop`
- `scale_out_zone`
- `runner_zone`

也就是說，系統給你的不只是方向，而是一整段交易管理劇本。

### 3. 風險控制

風控是這個專案不可刪的核心，主要包含：

- 高波動暫停
- 大波動 Telegram 提醒
- 逆勢方向暫停
- 連敗冷卻
- 消息面獨立提醒
- 支撐 / 壓力 / 破位後下一站地圖

這一層的目的不是讓你更常交易，而是讓你少犯不必要的錯。

### 4. 驗證與回饋

專案一定要保留 feedback loop，否則它只會退化成一般看盤工具。

目前驗證分兩種：

- `live` 成效追蹤
  - 看真實發送出去的 Telegram 事件後續表現

- `historical replay`
  - 看長時間歷史資料下，哪些事件真的有 edge

重點不是單看命中率，而是：

- 哪類事件值得做
- 哪類事件只值得看
- 哪類事件應該降級或停用

## 目前的實戰原則

### 該保留的原則

- 不追第一根
- 不把 `approach` 當進場
- 不把單日體感當成結論
- 不把消息面硬塞進技術決策
- 不把所有事件都視為對稱有效

### 該避免的方向

- 把專案做成「AI 猜方向工具」
- 把所有提示都包裝成可交易訊號
- 為了增加訊號而放鬆事件定義
- 用更多指標掩蓋沒有 edge 的事件

## 對功能取捨的判準

之後新增或修改功能時，只問四件事：

1. 它有沒有強化進場？
2. 它有沒有改善出場？
3. 它有沒有補強風控？
4. 它有沒有提升驗證能力？

如果四者都沒有，就不該加進專案。

## 現階段最重要的維護方向

- 維護事件分層，不讓所有提醒重新混成一團
- 維護 live 成效追蹤，持續看哪些事件真的有用
- 維護 protection layer，確保壞盤不要亂做
- 維護 Telegram 可讀性，讓通知直接對應實戰決策

## 總結

`check_price` 的價值，不在於它像不像一個會喊單的 AI。

它真正的價值在於：

- 把市場拆解得比人穩定
- 把決策流程寫得比人一致
- 把回顧與修正做得比人完整

所以這個專案應該持續被打磨成：

> 散戶可用的 AI 交易決策系統

而不是：

> 一個看起來很聰明、但沒有驗證閉環的預測工具
