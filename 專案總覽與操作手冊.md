# check_price 專案總覽與操作手冊

## 先看定位

`check_price` 是一套加密市場決策輔助系統，不是全自動交易機器人。

它的工作是：

- 整理多時間框架盤面
- 生成可執行的價格地圖
- 依事件層級發送 Telegram 提醒
- 持續追蹤 live 成效與歷史 replay

## 目前事件層級

這個專案最重要的不是單一方向，而是事件分層。

### `approach_up / approach_down`

- 戰備層
- 代表價格接近關鍵區
- 用來提醒你先看價位、止損與目標
- 不代表立刻下單

### `breakout_touch_up / breakout_touch_down`

- 試單層
- 代表第一次碰到關鍵位
- 仍不算完成確認

### `effective_long_breakout / effective_short_breakdown`

- 首層確認
- 代表市場出現第一層可交易條件

### `retest_hold_*`

- setup 層
- 代表劇本延續，但不保證是最佳首次進場點

### `second_breakout_* / second_breakdown_*`

- 保守掛單 / 確認加碼層
- 目前已帶：
  - `RR(TP1)`
  - `order_type`
  - `cancel_after_minutes`
  - `quality`

## 你應該怎麼使用這套系統

### 日常節奏

1. 先看 4 小時摘要，知道今天的大區間與上下破價。
2. 收到 `approach` 時，進入戰備，不追價。
3. 收到 `breakout_touch` 時，知道市場開始試方向，但還沒完成確認。
4. 收到 `effective` 時，再考慮是不是第一層進場候選。
5. 收到 `retest / second_*` 時，判斷是否保守掛單或確認加碼。

### 目前比較合理的實戰理解

- `approach`
  - 用來準備，不用來開單

- `breakout_touch`
  - 用來提醒第一次測試，不用當成完成確認

- `effective`
  - 用來看第一層確認是否成立

- `retest`
  - 用來看劇本是否延續

- `second_*`
  - 用來保守掛單或確認加碼

## 常用指令

### 跑單次分析

```powershell
python shadow_mode.py BTC ETH XRP SOL --llama off
```

### 啟動本地查詢頁

```powershell
python advisor_api_server.py --host 127.0.0.1 --port 8765 --llama off --risk-profile conservative
```

### 單輪測試提醒 daemon

```powershell
python market_alert_daemon.py --config watchlist.json --state-db alert_state.db --interval-seconds 60 --llama off --once
```

### 單輪產生 4 小時摘要

```powershell
python hourly_wait_reporter.py --config watchlist.json --once --llama off
```

### 重跑事件成效報告

```powershell
python alert_performance_tracker.py --db-path alert_state.db --report-md 提醒事件成效報告.md --loop-minutes 0
```

### 重跑歷史 replay

```powershell
python historical_replay_backtest.py --config watchlist.json --start-date 2025-12-20 --end-date 2026-03-20
```

## 現階段最重要的維護方向

- 維護事件分層，不讓所有訊號重新混在一起
- 維護 protection layer，避免壞盤亂做
- 維護 live 成效報告，持續看哪些事件真的有 edge
- 維護 Telegram 可讀性，讓通知直接對應到實戰流程

## 一句話總結

這個專案的重點不是猜漲跌，而是把交易流程變成：

- 可以看懂
- 可以執行
- 可以回頭驗證
- 可以持續修正
