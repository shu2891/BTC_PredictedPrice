# 策略設計與演進筆記

更新時間：2026-03-17

## 設計目標

這套專案不是自動下單器，而是：

- 幫短線交易者快速辨識市場結構
- 提供可執行的上破 / 下破價位與止損
- 避免在區間中間亂追
- 保留後續回填資料，讓策略能持續修正

## 已落地的策略骨架

### 多時間框架

- `5m`：觸發與短線方向
- `4h`：主方向濾網
- `1h / 24h`：波動、趨勢與風險補充

### 主要訊號

- `long / short / neutral`
- `大區間`
- `短線執行區`
- `上破價 / 下破價`
- `有效突破 / 有效跌破`

### 核心條件

- EMA 快慢線
- RSI
- VWAP 相對位置
- 量能比
- 假突破過濾

## 報告演進重點

### 第一階段

- 從單純 `watch / avoid` 升級成有提醒價位
- 加入區間、時間窗、上破 / 下破價

### 第二階段

- 改成偏短線框架
- 加入 `long / short / neutral`
- 把價位拆成：
  - `大區間`
  - `短線執行區`
  - `中間觀望區`

### 第三階段

- 報告改成更交易導向
- 加入：
  - `上破價 / 止損 / 現貨止損%`
  - `下破價 / 止損 / 現貨止損%`
- 每小時 Telegram 摘要改為中文精簡版

### 第四階段

- 價格提醒改成兩段式：
  - `touch`
  - `effective`

### 第五階段（試驗版）

- 加入 `market_regime`
- 在強單邊時提高逆勢訊號門檻，但不封印逆勢能力

## 為什麼不直接封印做空或做多

因為市場 regime 會輪動：

- 多頭月份裡，空方訊號容易失效
- 空頭月份裡，多方訊號也可能同樣失效

所以比起固定禁止一邊，更合理的是：

- 保留雙向能力
- 只在 `bull_trend / bear_trend` 時提高逆勢門檻

## 目前對實戰最有幫助的欄位

- `上破價`
- `上破止損`
- `上破現貨止損%`
- `下破價`
- `下破止損`
- `下破現貨止損%`
- `短線執行區`
- `market_regime`
